# DEBUGGING-NOTES.md — สมุดจด error ของฉัน

> ไฟล์นี้คือ "เพื่อนคู่ใจตอนเจอ error" — เปิดดู cheatsheet ด้านล่างเวลาเจอ error สีแดง ๆ แล้วจดทุกตัวที่เคยเจอไว้ในตารางท้ายไฟล์ จะได้กลับมาเปิดดูได้เมื่อเจอซ้ำ
> ใช้คู่กับ **Mini-Phase A — Debugging & DevTools** (แทรกหลัง Phase 2)

---

## 😱 เจอ error อย่าเพิ่งตกใจ

error เป็นเรื่องปกติของการเขียนโค้ด **ทุกคน** เจอทุกวัน รวมถึงโปรแกรมเมอร์มืออาชีพด้วย — error ไม่ใช่ความล้มเหลว แต่เป็น **เบาะแสว่าต้องแก้อะไรตรงไหน** ทักษะที่สำคัญที่สุดอย่างหนึ่งของ developer คือ "อ่าน error ออก" ไม่ใช่ "ไม่เคยเจอ error"

---

## 🔎 4 ขั้นอ่าน error (Cheatsheet)

เจอ error เมื่อไหร่ ทำตาม 4 ขั้นนี้ตามลำดับ อย่าข้าม:

### 1️⃣ อ่านข้อความบนสุดก่อน (อ่านบรรทัดแรก)

error ที่สำคัญที่สุดมัก **อยู่บรรทัดบนสุด** — บอกว่า "เกิดอะไรขึ้น" เป็นภาษาอังกฤษ ลองอ่านช้า ๆ ทีละคำ ยังไม่ต้องสนใจข้อความยาว ๆ ข้างล่าง คำที่เจอบ่อย:
- `is not defined` — เรียกใช้ตัวแปร/ฟังก์ชันที่ยังไม่มี หรือสะกดชื่อผิด
- `is not a function` — เรียกของที่ไม่ใช่ฟังก์ชันเหมือนเป็นฟังก์ชัน
- `Cannot read properties of undefined` — ไปหยิบค่าจากของที่ยังเป็น `undefined` (ของยังไม่มา/ยังไม่ได้กำหนด)
- `Unexpected token` — พิมพ์ syntax ผิด (เช่น ลืมวงเล็บ ลืมปีกกา พิมพ์ตกหล่น)
- `Module not found` — import ไฟล์/แพ็กเกจที่หาไม่เจอ (พิมพ์ path ผิด หรือยังไม่ได้ `npm install`)

### 2️⃣ ดูว่า error ชี้ไปที่ไฟล์ไหน บรรทัดที่เท่าไหร่

error มักบอก **ชื่อไฟล์ + เลขบรรทัด** (เช่น `TaskList.jsx:23`) — ไปเปิดไฟล์นั้น เลื่อนไปบรรทัดนั้น แล้วดูว่ารอบ ๆ บรรทัดนั้นเขียนอะไรไว้ จุดที่ error ชี้คือจุดเริ่มต้นที่ควรสำรวจก่อน

### 3️⃣ ไล่ดู stack trace (รายการบรรทัดยาว ๆ ข้างล่าง)

ข้อความยาว ๆ ที่ไล่ลงมาด้านล่างเรียกว่า **stack trace** — มันคือ "เส้นทางย้อนรอย" ว่าโค้ดวิ่งผ่านที่ไหนมาบ้างก่อนจะพัง อ่านจากบนลงล่าง มองหา **บรรทัดที่เป็นไฟล์ของเราเอง** (ไม่ใช่ไฟล์ใน `node_modules`) เพราะปัญหามักอยู่ในโค้ดของเรา ไม่ใช่ในแพ็กเกจของคนอื่น

### 4️⃣ ลองแก้ / ค้นหา

พอเข้าใจแล้วว่าเกิดอะไร ที่ไหน:
- **ลองแก้จุดที่สงสัยก่อน** (เช่น แก้ชื่อที่สะกดผิด เติมวงเล็บที่ขาด) แล้วรันดูใหม่
- ถ้ายังไม่หาย **ก๊อปข้อความ error บรรทัดบนสุดไปค้นหา** (Google / ถาม Claude) — บอก Claude ว่า "เจอ error นี้ ช่วยอธิบายว่าเกิดจากอะไร อย่าเพิ่งแก้ให้ ขอเข้าใจก่อน"
- ใช้ `console.log()` แทรกดูค่าตัวแปรระหว่างทาง ว่าค่าจริงเป็นอะไร ตรงกับที่คิดไหม

> 💡 จำไว้: เปลี่ยนทีละอย่าง แล้วรันดูทีละครั้ง — อย่าแก้หลายจุดพร้อมกัน เพราะถ้าหายจะไม่รู้ว่าจุดไหนที่แก้ได้ผล
